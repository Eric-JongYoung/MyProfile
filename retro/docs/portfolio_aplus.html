✅ 상세내용

Kosta 에서 교육을 받으면서 처음으로 진행하였던 web 프로젝트 이며, Jsp & Servlet 기반으로 제작한 프로젝트 입니다.

현대인들은 PC보다 스마트폰을 더 자주 구매하고 사용한다. 이러한 니즈를 만족시키기 위해서 일상에서 친숙한 스마트 기기들을 판매 대상으로 선정하였다.
애플 스토어 공식 사이트 디자인의 세련됨과 유려함은 가져가면서 UI의 불편함을 개선하여 사용자의 편의성과 접근성을 높여서 판매실적을 향상 시킨다.

🛠️ 개발환경

  • Java 11
  • Oracle 19c
  • Jsp & Servlet
  • Mac OS Ventura 13.0.1, Windows 10
  • intelliJ IDEA, Eclipse, DBeaver, SQL Developer
  • Git & GitHub
  • Notion

👾 담당 기능

역할 : 팀장
  • [제품 리스트]
    • 제품이 db를 바탕으로 front에서 나오도록 기능구현
    • 회원이 제품 상세페이지 기능구현
    • 관리자가 관리자페이지에서 제품을 등록하면 화면에 표현되고 회원이 구매가능하도록 기능 구현
  • [관리자페이지]
    • 관리자가 제품을 등록,수정,삭제 가능하도록 기능 구현
    • 회원이 주문한 내역을 조회 할 수 있도록 기능 구현
  • [고객센터]
    • 회원만 고객센터에 질문 가능 하도록 기능구현
    • 관리자만 회원이 남긴 질문글에 답변을 달 수 있도록 기능 구현

💡 깨달은점

  • - Web 프로젝트를 Back-End 에서 부터 View 까지 처음 만들어본 프로젝트로써, Back-End 와 View 연결의 중요성을 깨달음
  • - 해당 프로젝트를 통하여 MVC 구조를 조금더 이해하게 됨.
  • - 저의 MacOS와 팀원들의 Windows 간에 협업이 이루어질때, 초기에 환경세팅의 중요성을 깨달음.

🔥 보완해야할 점

  • - 각종 정보 입력시 정규화에 대한 필요성
  • - 각 method 흐름 중에 대한 검증단계의 필요성
  • - 팀원마다 각자 Back-End에서 View까지 진행함으로써, BackEnd의 코드와 View의 모양이 각자 다른 모양으로 생성되는것을 봄으로써, 다음 프로젝트일때는 일관되게 초기세팅을 하고 Back-End는 interface 기반으로 같이 설계하고, View는 처음부터 같은 Bootsrap Template를 사용해서 일관성을 맞추고 싶다는 생각이 들었다.
